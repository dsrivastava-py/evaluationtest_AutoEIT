## 📌 Project Overview
This repository contains my submission for **Specific Test I (Audio file transcription)** for the AutoEIT project. The goal of this task was to process raw Elicited Imitation Task (EIT) audio files in Spanish, transcribe the utterances with high accuracy, and align them to the target stimuli while strictly adhering to the "maintenance of meaning" rubric (preserving participant errors while correcting ASR phonetic errors).

## 📁 Repository Contents

* `AutoEIT_Transcriptions_FILLED.xlsx`: The final, cleaned deliverable containing the exact production transcripts for the four participants (38010, 38011, 38012, 38015) in Column C.
* `AutoEIT_Task_1.ipynb`: The master Jupyter/Colab notebook containing the end-to-end automated transcription, sequence alignment, and evaluation pipeline.
* `AutoEIT_Full_Transcription_Report.csv`: A detailed analytical report generated by the pipeline, documenting confidence intervals, Levenshtein edit distances, and Whisper log probabilities for every transcribed segment.

## ⚙️ Methodology & Pipeline Architecture

To process the long-form EIT audio files efficiently and accurately, I engineered an automated, data-driven pipeline:

1.  **VAD-Enabled Transcription (`faster-whisper`):** I utilized the `faster-whisper` library (large-v3 model) configured with strict Voice Activity Detection (VAD). This was crucial to programmatically bypass the initial 2.5 minutes of English instructions and prevent the ASR model from entering hallucination loops during the long participant listening pauses.
2.  **Dynamic Sequence Alignment:** Because Whisper outputs time-stamped segments based on breath groups rather than logical sentences, I engineered a fuzzy string matching script using Levenshtein distance (`thefuzz`). This automatically aligned the arbitrary ASR segments to the 30 specific target stimuli required by the EIT protocol.
3.  **Confidence & Anomaly Scoring:**
    The pipeline calculates a confidence metric (`[HIGH]`, `[MEDIUM]`, `[LOW]`, `[SILENT]`) for each row based on the Edit Distance to the target stimulus and Whisper's internal `AvgLogProb`. 

## 🔍 Evaluation Strategy & QA

The AutoEIT scoring rubric requires strict adherence to exact production—meaning participant grammatical errors must be preserved, while ASR phonetic misinterpretations must be corrected. 

As a non-native Spanish speaker, relying solely on manual acoustic evaluation to catch filtered disfluencies presented a constraint. To solve this, I relied on my discrepancy detection pipeline:
* I used the generated `AutoEIT_Full_Transcription_Report.csv` to isolate anomalies (segments flagged as `[LOW]` confidence or exhibiting high edit distances).
* By performing differential text analysis on these flagged sequences, I could systematically identify and resolve structural anomalies (e.g., cases where the ASR stitched overlapping audio segments together).
* A final manual QA sweep was performed on the output data to delete ASR hallucinations and format the final `.xlsx` file cleanly, ensuring the transcripts accurately reflect the acoustic reality without over-normalizing the participants' global language proficiency.

## 🚀 How to Run the Code
1. Open `AutoEIT_Task_1.ipynb` in Google Colab (recommended for free T4 GPU access).
2. Upload the raw audio files (`.mp3` or `.wav`) and the empty target stimuli CSV.
3. Run the cells sequentially to install dependencies, generate the base ASR transcriptions, execute the fuzzy alignment sequence, and export the final analytical report.
